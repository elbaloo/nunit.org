Planned Features

The following table lists features we plan to implement in version 1.0 of NUnitLite. Planned releases may well be combined or others may be added. Features may be added or dropped. In particular, see the next table, which lists features being considered.

After the 1.0 release, we plan to merge the source code of NUnit and NUnitLite and use conditional compilation to produce each assembly. However, if NUnit 3.0 development proceeds more rapidly than expected, this merging could take place before the 1.0 release, but is not expected to impact the availability of features in NUnitLite.
FeatureReleaseStatus
Support for .NET 3.0, 3.5, and 4.0, .NET CF 3.5 and 4.0? and corresponding Mono releases. 0.6 The existing implementations are expected to work without problem on newer versions but changes may be needed to take advantage of new features.
Allow use of static methods for tests, SetUp, TearDown, TestFixtureSetUp and TestFixtureTearDown. 0.6
PlatformAttribute 0.6
CategoryAttribute 0.6
Test runner should produce Xml output using a defined schema. Initially the schema may be different from that used by NUnit. A common schema is planned for a future NUnit release. 0.7
Test runner should send final output (text or xml) to a TCP endpoint provided by a desktop runner program. 0.7
Standalone app for deploying and running tests and displaying results sent via TCP 0.7
Send incremental result packets for each test via TCP, while the test run goes on. Desktop runner should show results progressively and save the xml file for the full run. 0.8 Requires defining a format, which might be SOAP, custom XML, JSON or something entirely different.
Standalone desktop gui to view NUnitLite xml results. 0.8 Based on existing NUnit viewer. NUnit itself may do this in the future.
Support for Linux devices running Mono 0.9 Since most Linux-based devices run a fairly complete Mono implementation, this is expected to work without further effort. It needs testing, however.

Additional Features

This table lists features under consideration for inclusion in a future release of NUnitLite as well as those that may be implemented separately from NUnitLite itself.
FeatureReleaseStatus
Report each successful Assert in addition to failures Could be useful in some heavily validated domains, such as medical
Report errors without use of exceptions either as a run-time or a compilation option Some platforms or environments are not friendly to Exceptions
Recognize and run NUnitLite tests in NUnit, with support for all features. This implies an add-in for NUnit in order to recognize inheritance-based tests and to apply NUnitLite semantics to the test run.
Run and display results in a simple Gui runner on the device For serious project work, this is probably not needed. However, it's easy to do and may be useful for testing.
Use NUnit to deploy and run tests on a device receiving results via TCP as the test progresses Not expected before the NUnit 3.0 release
NAnt task for executing NUnitLite tests NUnit task may be modified to do this
MsBuild task for executing NUnitLite tests Could be part of an NUnit task
VS2005/2008 add-in for running NUnitLite tests In each case, this feature could be part of an NUnit add-in
SharpDevelop add-in for running NUnitLite tests
MonoDevelop add-in for running NUnitLite tests
Add-in testing support for Visual Studio. In each case, this will likely be in the form of an add-in that tests other add-ins.
