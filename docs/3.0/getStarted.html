Getting Started with NUnit 3.0

To get a copy of the NUnit 3.0 Alpha release, you can use various installation approaches.

  1. Download a zip file containing all the NUnit components.
  2. Install the NuGet packages for the full framework.
  3. Install the NuGet nunitlite package.
  4. Combined Approach

Downloading the Zip File

Download the latest binary zip from our Download page, from GitHub or right here. Unzip the file into any convenient directory.

If you would like be able to run nunit-console from the commandline, put the bin directory, containing nunit-console.exe on your path. If you also need to run an older version of nunit-console from the commandline on the same machine, rename the new nunit-console.exe to something like nunit3-console.exe.

In your test assemblies, add a reference to nunit.framework.dll or nunitlite.dll, using the copy in the subdirectory for the appropriate runtime version. For example, if you are targeting .NET 4.0, you should reference the framework assembly in the net-4.0 subdirectory.

Run nunit-console from the command line, giving it the path to your test assembly. To run NUnit's own framework tests from within the NUnit bin directory, enter

            nunit-console net-2.0/nunit.framework.tests.dll

Using NuGet Packages for the Full Framework

In VisualStudio, from the Tools menu, select NuGet Package Manager | Manage NuGet packages for solution...

Since NUnit 3.0 Alpha is a prerelease, ensure that the lefthand combobox at the top of the Manage NuGet Packages dialog indicates "Include Prerelease." Scroll or use search to locate the NUnit Version 3 and NUnit.Runners packages. Install both packages. The NUnit Version 3 package should be referenced by each of your test assemblies, but not by any others.

Locate nunit-console in the packages\NUnit.3.0.0-alpha\tools directory under your solution. This is the location from which you must run nunit-console when using this approach. We recommend you only use this approach when running under the control of a script on your build server.

Using NuGet NUnitLite Package

In VisualStudio, from the Tools menu, select NuGet Package Manager | Manage NuGet packages for solution...

Since NUnit 3.0 Alpha is a prerelease, ensure that the lefthand combobox at the top of the Manage NuGet Packages dialog indicates "Include Prerelease." Scroll or use search to locate the NUnitLite Version 3 and install it. The package should be referenced by each of your test assemblies, but not by any others.

As a result of your installation, a file named "Program.cs" will have been copied into each of your test projects. NUnitLite test projects are console applications and this file contains the Main() program for your tests. If you already have a Main() in some other class, you may delete the file. This file also contains comments describing how the main should call the built-in NUnitLite runner.

To run your tests, simply run your executable test assembly. No other runner is needed.

Combined Approach

This approach is useful if you would like to use a single copy of nunit-console with individual copies of the framework in each project.

Simply follow the zip file procedure to get a central copy of NUnit on your system. Then install the NUnit Version 3 NuGet package in each of your test assemblies. For desktop use by developers, this approach may give you the best of both worlds.
