RequiresMTAAttribute (NUnit 2.5)

The RequiresMTAAttribute is used on a test method, class or assembly to specify that the tests should be run in the multi-threaded apartment. When running tests in parallel, the test is simply scheduled to execute from a queue that uses the MTA. When the parallel feature is not in use, it causes creation of a new thread if the parent test is not already running in the MTA. Note that MTA execution is the default for NUnit so this attribute is not normally specified.

Examples


// All the tests in the assembly will run in the MTA by default
[assembly:RequiresMTA]

...

// TestFixture requiring a separate thread
[TestFixture, RequiresMTA]
public class FixtureRequiringMTA
{
  // An MTA thread will be created and all
  // tests in the fixture will run on it
  // unless the containing assembly is
  // already running on an MTA Thread
}

[TestFixture]
public class AnotherFixture
{
  [Test, RequiresMTA]
  public void TestRequiringMTA()
  {
    // A separate thread will be created for this test
    // unless the containing fixture is already running 
    // in the MTA.
  }
}
