NUnit 2.4.7 - March 30, 2008

General

  • NUnit no longer uses log4net for its internal logging, eliminating conflicts with tested applications that used an earlier version of log4net.

Gui

  • The level of logging sent to the gui for display is now controlled by a setting in the config file for each test. If no setting is provided, the default value is Error. See the NUnitTests.config file for an example of how to set this.

Extensibility

  • NUnit now includes the RowTest extension, written by Andreas Schlapsi, in it's extension assemblies. This extension allows you to write test methods that take arguments and to provide multiple sets of argument values using the RowAttribute. To use RowTest, your test must reference the nunit.framework.extensions assembly.

    Note: Merging extensions into NUnit's own extension assembly is an experiment we are trying for this release. The approach may change in future releases.

Bug Fixes

  • Fixed null reference exception in SimpleSettingsDialog
  • String equality comparisons now use Equals rather than String.Compare, which was causing some unexpected results.
  • Fixed a race condition in the EventPump
  • Exceptions in Dispose are now handled properly
  • Projects in directories for which the user does not have write access are closed correctly.
  • The Reload command is now disabled when no project is open, eliminating a crashing problem.
  • Visual Studio projects with platforms other than Any CPU may now be opened correctly.
  • Assemblies located on a UNC path now load correctly
  • The EventListeners extension point is correctly initialized
  • Has.None no longer causes a null reference exception
  • Use of log4net for internal logging has been eliminated due to performance and compatibility problems.
